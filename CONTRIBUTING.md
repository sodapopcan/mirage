# Contributing

Contributions are very welcome!  There are just a few things:

1. If you're adding a feature, you might want to open an issue to discuss first.
1. Please include thorough tests.
1. Open one PR per feature.
1. While using LLMs is totally cool, **commits must be authored by you.**  CI
   will reject any commits made by LLMs.
1. Please DO NOT use "conventional commit" messages.  I otherwise don't care
   about your commit message style.
